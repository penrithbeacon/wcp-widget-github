About this widget

The GitHub WCP widget gives you a live view of your GitHub repositories inside Penrith Beacon WCP Dashboard. It calls the GitHub REST API directly using your own Personal Access Token — no data passes through any third party.

The widget exposes three components, each sized at 12 × 6 by default. That's a full stave in standard Penrith Beacon WCP layout. The intent is that you create a dedicated "GitHub" orchestration with staves for the components you use most, and switch to it via the Orchestration Manager when you want to browse your repositories.

Repositories
All your GitHub repos sorted by last push. Name, language, visibility, archived status, and an Open button per card.
Settings
Enter and manage your Personal Access Token. Shows the connected GitHub account once saved.

Setup Guide

You need a GitHub Personal Access Token (PAT) with the right scopes before the widget can show your repositories.

1. Create a Personal Access Token (classic)

  1. Go to github.com/settings/tokens
  2. Click Generate new token → Generate new token (classic)
  3. Give the token a descriptive name (e.g. "Penrith Beacon WCP")
  4. Set an expiry that suits your security policy (90 days recommended)
  5. Select the required scopes:
    • repo — full access to public and private repositories (recommended)
    • Or public_repo — public repositories only, if you don't need private repo access
  6. Click Generate token
  7. Copy the token immediately — GitHub only shows it once
Fine-grained tokens. If you prefer GitHub's newer fine-grained tokens, select Repository access → All repositories and grant the Contents: Read-only and Metadata: Read-only permissions. Note that fine-grained tokens may not cover all repository types depending on your account.

2. Configure the widget

Open the Settings component, paste your token into the Personal Access Token field, and click Save Settings. The connected account name will appear in the status bar once the token is verified. The Repositories component will then show your live data.

FAQ

Where is the token stored?

Your token lives in /app/data/config.json inside the container's named Docker volume (gh-data). It never leaves your machine except in API calls direct to GitHub.

How fresh is the data?

Repository data is cached for 60 seconds per instance. Click the Refresh button to bypass the cache immediately.

What are the GitHub API rate limits?

Authenticated requests have a limit of 5 000 requests per hour. Listing all repositories for a typical account uses 1–3 requests; you are very unlikely to hit the limit under normal usage.

Can I use this with a GitHub organisation?

Yes. The widget calls GET /user/repos?type=all, which returns personal repos plus organisation repos that you have access to under the authenticated account. Private org repos require the repo scope.

Can I run multiple instances pointing at different GitHub accounts?

Yes. Each WCP dashboard instance sends a Wcp-Instance-Id header; the widget stores separate credentials per instance in the same Docker volume.

What if my token expires or leaks?

Revoke it immediately from github.com/settings/tokens, generate a new one, and paste it into the Settings component.
